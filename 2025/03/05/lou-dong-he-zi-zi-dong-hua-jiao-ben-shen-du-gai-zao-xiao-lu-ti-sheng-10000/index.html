漏洞盒子自动化脚本深度改造，效率提升10000%！

2月份折腾了好一会，也是成功挖到260+漏洞，可问题是，这些漏洞该怎么提交呢？

写报告、搜集公司归属、整理渗透图片…一个漏洞至少折腾10分钟，260+个漏洞就要花费43个小时。

按每天工作8小时计算，差不多要连续奋战5.375天😅

好在之前我就已经找到了一个《漏洞盒子自动化脚本3.0》，还没用上呢，最近查找的时候，发现又多了一个漏洞盒子批量提交脚本5.1.26 - FreeBuf网络安全行业门户 ok，那么开始使用。

修改前——脚本的不便

但这个脚本用起来，并没有我想象中的那么美好😭：

  • 每一个厂商信息需要手动查询：所属地区跟行业的处理浪费不少时间。[每漏洞/2~3分钟]
  • 漏洞图片需要自己截图：并且还要放在指定文件夹中。文件多起来后，管理起来相当痛苦。[每漏洞/1~3分钟] (而且文件越多越麻烦)
  • 太多需要手动处理的杂事：权重需要自己查，有权重的还需要单独处理；对XML格式没有兼容，直接上传会转义…

于是，我决定在这个脚本的基础上，加入”一些东西”，经过N个版本的迭代，总算是做出一款只需要10~20分钟的半自动化部分，就能处理任意漏洞量的”效率更高的半自动化脚本”

魔改特色

  • 图片自动处理：利用Pillow库与playwright库，实现全自动图片生成和处理。

2~3分钟就能处理700+张照片，效率提升70000%！图片越多，提升效果越明显。

  • 工作流程化：批量收集信息、批量处理信息，流水线作业提升效率。
  • 智能匹配系统：使用cpca库与fuzzywuzzy库，自动匹配所属地区和行业，让繁琐的分类工作变得轻松。
  • 修复兼容问题：解决了复现过程中对<>符号的转义问题，不再出现格式混乱。
  • 异步处理与纠错机制：在提升处理速度的同时，增加了容错能力，让脚本更加稳定可靠。

流程介绍

①：准备漏洞基础信息

首先，需要在mid-bug表中填入漏洞的基本信息。必需的信息包括：URL（漏洞点的url）、请求包、响应包、漏洞类型。

  • 如果是sql注入，把影响参数填一下
  • “漏洞类型”指：sql注入、xxx公司xxxx处存在xxx漏洞，只要自己能识别就行

推荐：使用Yakit的WebFuzzer进行发包（这样才有请求包与响应包），然后在History中，搜索并筛选出命中漏洞同时导出。

②：配置基本信息

填写config.yaml，确定漏洞的基本参数。最重要的一点是，确定这个漏洞需要提交几张图片。如果需要调整，记得在photo函数中同步修改代码。

默认情况下，系统会生成这3张图：归属图、漏洞主页图、poc图

③：自动生成基础数据

运行main-产出表.py，通过basic_process函数处理，让mid-bug表自动获得漏洞id、漏洞主页和根域名信息。

④：获取网站信息

此时会弹出指引，要求使用[爱站网]，通过根域名，去得到权重、备案号、公司(False)，并手动添加到mid-bug中并进行处理。站长工具_网站综合批量查询_爱站网

  • 如果有权重，那么在特殊一栏填写权重。
  • 备案号为空的行，直接删除。
  • 如果有重复的公司，只保留其中一行。
⑤：公司信息处理

确认后，程序会进入enscan函数，自动读取mid-bug表，并将公司信息输出到output_data文件夹；接着调用行业模糊匹配函数，处理最新的行业信息表。

⑥：手动校正地区信息

接下来，行业信息表还需要一点手动调整 —— 找到省市中标记为”未知”的部分，手工查询并更新。

⑦：合并数据

确认无误后，运行company_info函数，系统会自动将行业信息表与mid-bug表合并，为mid-bug表添加公司（True）、省市位置、粗分行业、细分行业等信息。

⑧：图片处理

进入photo函数阶段，这里的POC证明图可能需要根据漏洞类型调整所需信息。例如，对于延时注入漏洞，POC证明图中应该包含延迟(ms)信息。

比如最简单的，把一张固定的图片复制到所有文件夹中。

⑨：执行最终提交

系统会自动将带有权重标记的行提取到新表中，并从mid-bug表中删除这些行。检查config.yaml配置无误后，手动运行main-提交漏洞.py。

⑩：审核与提交

使用草稿模式检查是否有问题，确认无误后直接提交。

zero：清理临时文件

最后手动删除output_data中的旧的行业信息表与已提交过的images，保持工作区整洁。

别被这些步骤给吓到，看起来步骤很多，总共有11个步骤，但实际上，需要手动的地方很少，而且没有重复性工作。至于手动的部分，我在下面也单独列举并分析：

需要手动的地方

不会打断工作流的：

填写config.yaml 与 表中的漏洞基础信息。

[这一步不需要也没必要省略吧]

根据需求，手动调整处理图片的代码。

[默认的三张图片处理不了很多场景，自己调整代码是必须的]

手动删除output_data中的旧的行业信息表与提交过的images.

不自动删除，是因为有可能还会需要。而且不删除理论上不影响下一次漏洞的提交（除非下一次又挖到了同一家公司）

会打断工作流的：

通过爱站网批量查询，然后手动添加结果到文件中并进行处理。

[如果有API的话，这一步其实可以更自动。人工处理，根据爱站网响应速度，大约需要1~2分钟]

对行业信息表进行手动处理，把“未知”的省市位置进行手工查询。

行业信息查询，我这里直接使用的Enscan。Enscan是一个对爱企查、天眼查的爬虫。

然后对它的结果进行一个处理。而Enscan查出来是这样的：

注册地址：xx市xx区xx中路……金融中心T3栋写字楼41楼

经营范围一般项目：汽车零配件批发；汽车零配件零售；建筑材料销售；金属材料销售；化工产品销售（不含许可类化工产品）……

由于注册地址可能只有省份跟街道，直接跳过了市区，所以cpca库不一定能识别，必须要手动纠正。

如果要进行优化，那么可能要魔改ENSCAN，或者寻找新思路（我看到过一个使用高德地图api的），才能避免手工。

可以看到，会打断工作流的部分其实都有优化空间，不过基于时间和成本的考虑，我认为目前这个版本已经足够高效了。

async def capture_website(df):
    """

    """
    # 测试

    async with async_playwright() as p:
        # 启动浏览器（无头模式可设置为 False 查看界面）
        browser = await p.chromium.launch(headless=True)
        # context 相当于浏览器的配置文件
        context = await browser.new_context(viewport={'width': 1920, 'height': 1080},ignore_https_errors=True)

        # 并发控制（防止内存溢出）
        semaphore = asyncio.Semaphore(5)
       
        async def process_url(index, url, id, row):
            async with semaphore:
                page = await context.new_page()
                try:
                    save_dir = get_save_dir(row)
                    id_dir = f"{save_dir}/{id}"
                    fpath = f"{id_dir}/主页2.png"
                    if os.path.exists(fpath):
                        logging.info(f"已存在：{id}/主页2")
                        return
                    if not os.path.exists(id_dir):
                        logging.error(f"{id_dir}公司目录不存在！")
                        return
                    
                    @retry(stop_max_attempt_number=3,wait_fixed=2000)
                    async def pagego(url,timeout):
                        await page.goto(url, timeout=timeout)
                    try:    
                        await pagego(str(url), timeout=60000)
                    except Exception as e:
                        logging.error(f"网络请求错误: {e}")
                        raise
                    # 唯一文件名（避免覆盖）
                    await page.screenshot(path=fpath, full_page=True)
                    logging.info(f"截图成功：{url}")
                except Exception as e:
                    logging.error(f"截图失败【{url}】: {str(e)}")
                finally:
                    await page.close()

        # 批量创建任务
        tasks = []
        for index, row in df.iterrows():
            item = process_url(index, row['漏洞主页'], row['漏洞id'], row)
            tasks.append(item)

        # 并发执行并等待所有任务完成 *传递多个进程
        await asyncio.gather(*tasks)
        await context.close()
        await browser.close()  
        logging.info("主页截图任务已完成")


# 统一的图片生成函数
def create_text_image(text, output_path, font_size=20, line_spacing=5, padding=20, 
                     bg_color=(255, 255, 255), text_color=(0, 0, 0), max_width=70, 
                     min_width=800):
    """
    创建包含文本的图片
    
    参数:
        text: 要显示的文本内容
        output_path: 输出图片路径
        font_size: 字体大小
        line_spacing: 行间距
        padding: 内边距
        bg_color: 背景颜色，RGB元组
        text_color: 文字颜色，RGB元组
        max_width: 每行最大字符数
        min_width: 图片最小宽度
    """
    font_path = "simhei.ttf"  # 使用中文字体
    font = ImageFont.truetype(font_path, font_size)
    
    # 自动换行处理
    wrapper = textwrap.TextWrapper(width=max_width)
    wrapped_text = "\n".join(wrapper.fill(line) for line in text.split('\n'))
    
    # 计算图片尺寸
    draw = ImageDraw.Draw(Image.new('RGB', (0,0)))
    text_bbox = draw.multiline_textbbox((0,0), wrapped_text, font=font, spacing=line_spacing)
    text_width = text_bbox[2] - text_bbox[0]
    text_height = text_bbox[3] - text_bbox[1]
    
    img_width = max(text_width + 2*padding, min_width)  # 确保至少有最小宽度
    img_height = text_height + 2*padding
    
    # 创建画布
    img = Image.new('RGB', (img_width, img_height), color=bg_color)
    draw = ImageDraw.Draw(img)
    
    # 绘制文字
    draw.multiline_text(
        (padding, padding),
        wrapped_text,
        fill=text_color,
        font=font,
        spacing=line_spacing
    )
    
    # 保存图片
    img.save(output_path)

智能地区与行业匹配

功能简介&实现思路

这个模块是为了适应enscan而存在的，前文提到，enscan会爬取并返回不那么完美的注册地址跟经营范围,所以要对通过模块，对结果进行二次加工。

地址处理：通过cpca库，对省市进行识别，并根据漏洞盒子的菜单进行颗粒度对齐。

行业分类：通过fuzzywuzzy库，把enscan得到的经营范围与漏洞盒子提供的经营范围进行模糊字符串匹配。

# 地址解析函数
def parse_address(address):
    df = cpca.transform([address])
    if not df.empty:
        province = df.iloc[0]['省']
        city = df.iloc[0]['市']
        # 直辖市的特殊处理 再加上漏洞盒子的处理
        #if province.find("北京") or province.find("上海") or province.find("天津") or province.find("重庆"):  find找到返回0 找不到返回-1
        try:
            if  "市" in province:  
                city = province
                province = province.replace("市",'')
            else:
                province = province.replace("省",'')
                city = city.replace("市",'') if city else "未知"
        except Exception as e:
            print(f"发生未知错误: {e}")
            province = "未知"
            city = "未知"
        # 针对省份中含有自治区之类的
        if "自治" in province:
            province = province[0:2]

        province = f"\"{province}\""
        city = f"\"{city}\""
        #["广东","深圳"]
        return f"[{province},{city}]"
    return "未知地区"

批量信息采集与处理

功能简介&实现思路

提交漏洞不仅需要报告，还需要很多辅助信息。我的设计思路是通过一张表格/input/mid-bug.xlsx，集中管理所有必要信息。相比魔改前的版本，现在的表格中大部分字段都是自动化填充的，无需重复操作。

上文说到了，该工作流启动的基本条件是填写表中的字段：

URL（漏洞点的url），请求包，响应包，漏洞类型

工作流函数自动生成漏洞id,漏洞主页,根域名

然后根据根域名，手动去爱站网批量查询 权重、备案号、公司(False) 爱站网的数据不是最新的，所以这里的公司我标注的False，通过爱企查这样的网站才能查出公司(True)

工作流调用enscan得到粗略的公司信息，然后调用模糊匹配函数得到具体的公司信息。缺点是“未知”部分需要手动查询，原因在上文已经介绍。

然后工作流会将output/公司信息.xlsx 合并到mid-bug中，并用公司(True)覆盖公司(False)

最后工作流将带有权重的行删除，保存最终文件。并把带有权重的行 与 当前配置的config.yaml单独导出，以便后面手动提交给补天/edu平台。

虽然步骤看起来不少😅，但这已经是我优化N次的结果了。要实现完全不打断工作流，还需要进一步改进。

    def process_urls(self):
        # 添加新列

    def _get_vulnerability_id(self, url):
        # 解析URL
        parsed = urlparse(url)
        # 获取域名（如果有端口号，去掉:符号）
        domain = parsed.netloc.replace(':', '')

    def _get_homepage(self, url):
        # 解析URL
        parsed = urlparse(url)

        self.df['主办单位名称'] = company_df['企业名称'].values[:len(self.df)]
        
        # 添加新的三列  [:len(self.df)] 是一个切片操作，用于从 company_df 的列中提取与 self.df 行数相同数量的元素。
        self.df['省市位置'] = company_df['省市位置'].values[:len(self.df)]
        self.df['粗分行业'] = company_df['粗分行业'].values[:len(self.df)]
        self.df['细分行业'] = company_df['细分行业'].values[:len(self.df)]
        
        logging.info("至此,企业信息更新完成")

        
        def save_weight(self):
        # 找出所有标记为权重的行
        weight_df = self.df[self.df['特殊'] == '权重'].copy()
        
        if len(weight_df) == 0:
            logging.info("没有找到标记为权重的行")
            return
            
        # 从原表中删除权重行
        self.df = self.df[self.df['特殊'] != '权重']
        
        # 保存权重数据到新文件
        weight_file = f"output_data/权重-{self.df['漏洞类型'].iloc[0]}.xlsx"
        try:
            weight_df.to_excel(weight_file, index=False)
            logging.info(f"已将权重数据保存到: {weight_file}")
            shutil.copy("config.yaml",f"output_data/config-{self.df['漏洞类型'].iloc[0]}.yaml")
            logging.info(f"已将config.yaml复制到output_data目录下")
        except Exception as e:
            logging.error(f"保存权重数据时出错: {e}")
            return
            
        # 保存更新后的原始数据
        try:
            self.save(FILE_PATH)
            logging.info("已从原表中删除权重数据")
        except Exception as e:
            logging.error(f"更新原始数据时出错: {e}")

    """处理文本中的占位符替换"""
    try:
        # 匹配图片、URL网络位置、协议的正则
        pattern_img = r'\{images([0-9])\}'
        pattern_netloc = r'\{netloc\}'
        pattern_scheme = r'\{scheme\}'
        pattern_html = r'<code class=\\"Bash\\">(.*?)<\/code>'
        
        netloc = urlparse(bug_url).netloc
        scheme = urlparse(bug_url).scheme
        
        def replace_images(match):
            index = int(match.group(1)) - 1
            if 0 <= index < len(content_list):
                return content_list[index]
            return match.group(0)
            
        def escape_content(match):
            inner_content = match.group(1)
            escaped = inner_content.replace("<", "&lt;").replace(">", "&gt;")
            return f'<code class="\\Bash\\">{escaped}</code>'
        
        # 应用所有替换
        text = re.sub(pattern_img, replace_images, text)
        text = re.sub(pattern_netloc, netloc, text)
        text = re.sub(pattern_scheme, scheme, text)
        text = re.sub(pattern_html, escape_content, text, flags=re.DOTALL)
        
        return text
    except Exception as e:
        logger.error(f"文本替换处理异常: {e}")
        return text

功能简介&实现思路

由于在工作流中经常需要手动处理文件，而且需要进行的网络请求也挺多的，断点重连功能是必不可少的。

这里的实现比较简单，就是用一个文件记录每次运行时的执行状态，经过检查点时保存当前状态，如果完整运行那就删除该文件。

关键代码片段
def load_state():
    if os.path.exists(STATE_FILE):
        with open(STATE_FILE, 'r') as f:
            return json.load(f)
    # 如果文件不存在，创建一个初始状态文件
    initial_state = {"last_completed_step": None}
    with open(STATE_FILE, 'w') as f:
        json.dump(initial_state, f)
    return initial_state

def save_state(step):
    with open(STATE_FILE, 'w') as f:
        json.dump({"last_completed_step": step}, f)


if __name__ == "__main__":
    # 加载上次执行状态
    state = load_state()
    last_step = state["last_completed_step"]
    
    # 创建worksheet实例
    w = worksheet(FILE_PATH)
    
    # 根据上次执行状态决定从哪里开始
    if not last_step:
        # 然后进行备案号等处理,这里手动完成
        w.basic_process()
        w.wait_for_user_action("域名信息")
        save_state("domain_info")
        enscan()
        save_state("enscan")
    elif last_step == "domain_info":
        enscan()
        save_state("enscan")
    
    if last_step in [None, "domain_info", "enscan"]:
        w.wait_for_user_action("模糊匹配修正")  
        save_state("fuzzy_match")
    
    if last_step in [None, "domain_info", "enscan", "fuzzy_match"]:
        # 合并企业信息
        w.company_info()
        save_state("company_info")
        w.save(FILE_PATH)

    if last_step in [None, "domain_info", "enscan", "fuzzy_match", "company_info"]:
        # 截图
        photo_main()
        save_state("photo")

    if last_step in [None, "domain_info", "enscan", "fuzzy_match", "company_info", "photo"]:
        # 提取权重数据
        w.save_weight()
        save_state("weight")

    # 完成所有步骤后清除状态文件
    if os.path.exists(STATE_FILE):
        os.remove(STATE_FILE)
